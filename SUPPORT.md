# Support

This document explains how to get help with OpenSIN-AI projects and where to direct different kinds of requests.

## Getting Help

### For General Questions

- **GitHub Discussions** — the best place for questions that are exploratory, directional, or need community input.
- **Organization README** — start with the [OpenSIN-AI organization profile](https://github.com/OpenSIN-AI) to understand our mission and structure.

### For Setup and Onboarding

- **[dev-setup](https://github.com/OpenSIN-AI/dev-setup)** — contains all setup guides, GitHub workflow documentation, and team standards.
- **[GitHub Guides](https://github.com/OpenSIN-AI/dev-setup/tree/main/github)** — covers profile setup, organization usage, projects, discussions, and collaboration flow.

### For Bug Reports

- **Issues** — use the bug report template in the relevant repository.
- Include clear reproduction steps, expected vs actual behavior, and evidence.

### For Feature Requests

- **Issues** — use the feature request template.
- Describe the problem first, then the proposed solution.

### For Security Issues

- **Do not disclose publicly.**
- Use GitHub's private vulnerability reporting or contact the security team through private channels.
- See our [Security Policy](./SECURITY.md) for details.

## What to Include When Asking for Help

- Which repository or area you're working with
- What you're trying to achieve
- What you've already tried
- Relevant error messages, logs, or screenshots
- Your environment (OS, tooling versions, etc.)

## Response Expectations

- We aim to acknowledge requests within a reasonable timeframe.
- Complex issues may require additional information or discussion.
- Not every request will be prioritized immediately — we triage based on impact and urgency.

## Community Standards

All interactions in OpenSIN-AI spaces are governed by our [Code of Conduct](./CODE_OF_CONDUCT.md). Please read it before participating.
